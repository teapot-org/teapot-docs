# 2. Назначение и цели разработки

2.1. Назначение разработки
-------------------------

Назначением разработки является автоматизация процесса ведения и управления разработкой методом "Канбан". Канбан — метод управления разработкой, реализующий принцип «точно в срок» и способствующий равномерному распределению нагрузки между работниками. При данном подходе весь процесс разработки прозрачен для всех членов команды. Задачи по мере поступления заносятся в отдельный список, откуда каждый разработчик может извлечь требуемую задачу.

# 3. Требования к программному изделию

3.1. Требования к структуре системы
-----------------------------------

### 3.1.1. Общая структура системы

Teapot - это клиент-серверное WEB-приложение (сайт), позволяющее автоматизировать организацию, управление и непосредственно разработку проектов, используя метод "Канбан". Сам Канбан представляет собой виртуальную доску, на которую закрепляются виртуальные карточки с заданиями. Карточки на доске распределяются по спискам, соответсвующим определённым стадиям разработки проектов. Таким образом, основная возможность TPT - создание виртуальных канбан-досок и размещение на них виртуальных карточек.

Сайт должен состоять из следующих элементов:

1. Главная страница
2. Форма регистрации
3. Форма авторизации
4. Общий список пользователей
5. Общий список организаций
6. Личный профиль пользователя
7. Раздел управления организациями
8. Раздел управления проектами
9. Список канбан-досок

Помимо этого, приложение позволяет выстраивать иерархическую структуру пользователей и канбан-досок. Элементами этой структуры являются:

1. Пользователи;
2. Проекты;
3. Организации;
4. Виртуальные доски.

### 3.1.2. Пользователи

Пользователь - это зарегистрированное в системе лицо, использующее TPT для создания канбан-досок, проектов и организаций и управления ими. Имеет свой личный профиль.

### 3.1.3. Проекты

Проект - элемент иерархической структуры, отражающий предмет разработки системы и/или проведения какой-либо иной деятельности. Проект имеет название, описание, и состоит не менее, чем из одного участника. Внутри проекта можно размещать канбан-доски. Подробнее о функциональных характеристиках проекта см. п. 3.3.1 (**FE-8**, **FE-15** - **FE-17**).

### 3.1.4. Организации

Организация - это группа пользователей, имеющая собственное название и профиль. Внутри организации можно создавать проекты и размещать канбан-доски. Подробнее см. п. 3.3.1 (**FE-9**, **FE-12** - **FE-14**, **FE-17**).

### 3.1.5. Виртуальные доски

См. п. 3.3.1 (**FE-5** - **FE-7**).

### 3.1.6. Главная страница

Если пользователь не зарегистрирован (не авторизован) в системе, на главной странице ему предлагается зарегистрироваться (авторизоваться). Если пользователь вошёл в систему, на главной странице отображается его личный профиль.

### 3.1.7. Регистрация пользователя

Регистрация должна осуществляться с помощью заполнения соответствующей формы регистрации, состоящей из обязательных и необязательных полей.

Обязательные поля:

- Адрес электронной почты;
- Логин;
- Пароль;
- Подтверждение пароля.

Необязательные поля:

- Имя;
- Фамилия;
- Дата рождения;
- О себе;
- Аватар.

### 3.1.8. Авторизация пользователя

Форма авторизации состоит из двух полей:

- Логин (или электронная почта);
- Пароль.

### 3.1.9. Профиль пользователя

На странице с профилем пользователя должна находиться основная информация о пользователе, его канбан-досках, а также организациях и проектах, в которых он состоит или которыми руководит. Часть информации о себе пользователь может скрыть.

Общедоступная информация:

- аватар;
- имя;
- фамилия;
- графа «О себе».

Информация с настраиваемым доступом:

- список организаций, в которых состоит пользователь;
- список проектов, в кторых состоит пользователь;
- список личных канбан-досок.

3.2. Требования к разграничению доступа пользователей в системе
---------------------------------------------------------------

### 3.2.1. Администраторы

Администраторы имеют доступ ко всем функциям приложения, описанным в п. 3.3.1 без ограничений. Также Администраторы имеют доступ к редактированию информации о всех пользователях, их проектах и организациях.

### 3.2.2. Пользователи

#### 3.2.2.1. Незарегистрированные (неавторизованные) пользователи 
Незарегистрированные и неавторизованные пользователи имеют доступ к функциям **FE-1** и **FE-2** п. 3.3.1.

#### 3.2.2.2. Авторизованные пользователи
Авторизованные пользователи имеют доступ к функциям **FE-3**, **FE-5** - **FE-18** п. 3.3.1. с ограничениями, описанными ниже.

Общие ограничения для всех пользователей:
1. **FE-5** - ограничение на редактирование и/или удаление чужих канбан-досок;
2. **FE-6** - ограничение на редактирование и/или удаление чужих списков карточек;
3. **FE-7** - ограничение на редактирование и/или удаление чужих карточек;

Пользователи, состоящие в проектах, делятся по правам доступа в соответствии с назначенным статусом:

1. "Создатель" - пользователь имеет доступ к функциям **FE-5** - **FE-8**, **FE-15** - **FE-17** п. 3.3.1 со следующими ограничениями:
    1. **FE-17** - ограничение на изменение своего статуса;
2. "Владелец" - пользователь имеет доступ к функциям **FE-5** - **FE-8**, **FE-15** - **FE-17** п. 3.3.1 со следующими ограничениями:
    1. **FE-8** - ограничение на удаление проекта;
    2. **FE-16** - ограничение на исключение из проекта пользователя со статусом "Создатель";
    3. **FE-17** - ограничение на изменение статуса пользователя со статусом "Создатель";
3. "Участник" - пользователь имеет доступ к функции **FE-7** п. 3.3.1 со ограничением на удаление;
4. "Приглашён" - пользователь имеет доступ к функции **FE-18** п. 3.3.1.

Пользователи, состоящие в организациях, делятся по правам доступа в соответствии с назначенным статусом:

1. "Создатель" - пользователь имеет доступ к функциям **FE-5** - **FE-9**, **FE-13** - **FE-17** п. 3.3.1 со следующими ограничениями:
    1. **FE-17** - ограничение на изменение своего статуса;
2. "Владелец" - пользователь имеет доступ к функциям **FE-5** - **FE-8**, **FE-15** - **FE-17** п. 3.3.1 со следующими ограничениями:
    1. **FE-9** - ограничение на удаление организации
    2. **FE-16** - ограничение на исключение из организации пользователя со статусом "Создатель";
    3. **FE-17** - ограничение на изменение статуса пользователя со статусом "Создатель";
3. "Участник" - пользователь имеет доступ к функции **FE-7** и **FE-8** п. 3.3.1 со следующими ограничениями:
    1. **FE-7** - ограничение на удаление карточки из доски, принадлежащей "Создателю" и/или "Владельцу";
4. "Приглашён" - пользователь имеет доступ к функции **FE-18** п. 3.3.1;
5. "Подал заявку" - пользователь имеет доступ к функции **FE-10** п. 3.3.1;
6. Пользолватель без статуса имеет доступ к функции **FE-12** п. 3.3.1.

3.3. Требования к функциональным характеристикам системы
--------------------------------------------------------

### 3.3.1. Основные функции

**FE-1** Регистрация и авторизация пользователя.

**FE-2** Регистрация и авторизация администратора.

**FE-3** Редактирование личного профиля.

**FE-4** Блокировка пользователей.

**FE-5** Создание, просмотр, редактирование и удаление виртуальных досок.

**FE-6** Добавление, редактирование и удаление списков карточек виртуальных досок, а так же установка ограничения на максимальное кол-во карточек в списке.

**FE-7** Создание, просмотр, редактирование, удаление карточек. В свою очередь, в редактирование карточки входит:

1. перемещение карточек по спискам;
2. прикрепление участников;
3. добавление тэгов;
4. изменение цвета;
5. комментирование карточек;
6. форматирование (markdown) описания карточек;
7. прикрепление доп. материалов (мультимедиа, текстовые файлы, ссылки);
8. создание ярлыка карточки в других досках.

**FE-8** Создание, редактирование, удаление проектов, а так же создание виртуальных досок внутри проекта (FE-5, FE-6, FE-7).

**FE-9** Создание, редактирование, удаление организаций, создание проектов и досок внутри организации.

**FE-10** Просмотр списка пользователей

**FE-11** Просмотр списка организаций

**FE-12** Подача заявки на участие в организации

**FE-13** Приглашение участников в организацию

**FE-14** Добавление, исключение участников из организации

**FE-15** Приглашение участников в проект

**FE-16** Добавление, исключение участников из проекта

**FE-17** Назначение ролей (статусов) участникам организации или проекта

**FE-18** Принятие (отклонение) приглашения в организацию или проект
